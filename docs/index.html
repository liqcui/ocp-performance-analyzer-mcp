🔧 关键组件说明

1. MCP服务器 (FastMCP) - 基于Model Context Protocol的服务器，暴露15+分析工具供AI代理调用

2. AI代理 (LangGraph) - 智能代理使用OpenAI/兼容LLM，实现对话式分析和自动化报告生成

3. 工具收集器 - 通过PromQL查询Prometheus，收集51+ETCD性能指标

4. ELT管道 - 将原始JSON数据转换为结构化HTML表格，便于展示和分析

5. DuckDB存储 - 时序数据库存储历史性能数据，支持趋势分析

🎯 核心功能演示

1️⃣ 实时性能监控

功能: 通过Prometheus实时采集ETCD集群性能指标
覆盖指标: 51个ETCD核心指标，涵盖磁盘、网络、CPU、内存等
更新频率: 2分钟采样窗口，实时流式响应

2️⃣ AI对话式分析

# 用户查询示例 用户: "分析过去1小时ETCD的WAL fsync性能" # AI执行流程 1. 调用工具: get_etcd_disk_wal_fsync(duration="1h") 2. 收集指标: P50/P90/P99延迟分布 3. 分析数据: 对比阈值 (目标: P99 < 10ms) 4. 生成建议: 如果超标，提供优化方案 # 返回结果 📊 数据结果: - WAL Fsync P99: 8.2ms ✓ (优秀) - WAL Fsync P90: 5.1ms - WAL Fsync P50: 2.3ms 🤖 AI分析: WAL fsync性能优秀，P99延迟8.2ms远低于10ms阈值。 磁盘写入性能稳定，无需优化。

3️⃣ 自动瓶颈检测

检测维度:
• CPU使用率 (阈值: 警告70%, 严重85%)
• 内存使用率 (阈值: 警告70%, 严重85%)
• WAL Fsync延迟 (目标: P99 < 10ms)
• Backend Commit延迟 (目标: P99 < 25ms)
• 磁盘I/O Wait (目标: < 10%)
• 网络延迟 (目标: Peer < 50ms)

4️⃣ 完整性能报告

# 报告内容结构 ┌─────────────────────────────────────┐ │ 📋 ETCD性能分析报告 │ │ 时间范围: 2026-04-12 14:00 - 15:00 │ ├─────────────────────────────────────┤ │ 1. 执行摘要 │ │ • 集群状态: 健康 ✓ │ │ • 发现问题: 1个警告 │ │ • 优先级: 中等 │ ├─────────────────────────────────────┤ │ 2. 关键指标 │ │ ✓ WAL Fsync P99: 8.2ms │ │ ⚠ Backend Commit P99: 32.5ms │ │ ✓ CPU使用率: 45% │ │ ✓ 内存使用率: 58% │ │ ✗ 磁盘I/O Wait: 15% │ ├─────────────────────────────────────┤ │ 3. 瓶颈分析 │ │ 🔴 磁盘I/O瓶颈 │ │ - I/O Wait 15%过高 │ │ - 影响Backend Commit性能 │ ├─────────────────────────────────────┤ │ 4. 优化建议 │ │ 1. 使用更快的SSD (NVMe) │ │ 2. 调整I/O调度器为deadline │ │ 3. 检查磁盘竞争进程 │ └─────────────────────────────────────┘
